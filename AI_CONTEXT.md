# Spanish Vocab Bot — AI Context

## Назначение
Telegram-бот для изучения испанского языка с нуля (уровень A1-A2).
Объяснения и переводы — на русском языке.

## Техстек
- **Python 3.14**
- **python-telegram-bot >= 22.4** (с job-queue для напоминаний)
- **anthropic >= 0.40.0** — Claude Sonnet 4.6 для генерации контента
- **SQLite** — локальная база данных
- **Railway** — деплой, persistent Volume для БД примонтирован на `/data`

## Файлы
- `bot.py` — точка входа, все Telegram-хендлеры и команды
- `db.py` — вся работа с БД: схема, CRUD, логика интервалов
- `ai_helper.py` — промпты к Claude: объяснение слова, подбор частотных слов
- `requirements.txt`, `railway.json`, `Procfile` — деплой

## База данных (`words`)
Таблица `words`, одна на всех пользователей (различаются по `user_id`).

Ключевые поля:
| Поле | Тип | Назначение |
|------|-----|-----------|
| `phrase` | TEXT | Слово с артиклем для существительных (`la casa`), инфинитив для глаголов |
| `meaning` | TEXT | Перевод на русском |
| `part_of_speech` | TEXT | существительное / глагол / прилагательное / наречие / ... |
| `cefr_level` | TEXT | A1 или A2 |
| `examples` | TEXT | JSON-массив строк с примерами |
| `conjugation` | TEXT | Спряжение глагола в настоящем времени (только для глаголов) |
| `interval_stage` | INT | Текущая ступень кривой повторений (0–6, 6 = mastered) |
| `next_review_date` | TEXT | ISO-дата следующего повторения |
| `status` | TEXT | collected / learning / familiar / active / mastered / skipped |
| `pool` | TEXT | scheduled / overdue |
| `added_window` | TEXT | morning / evening — окно суток при добавлении |

## Интервальные повторения
**Кривая:** 1 → 3 → 7 → 14 → 30 → 90 дней (6 ступеней, `INTERVALS` в `db.py`)

**Первое повторение** зависит от времени суток (московское время, UTC+3):
- Добавлено утром (6–14) → первое повторение сегодня же
- Добавлено вечером (14–23) или ночью → первое повторение завтра

**Оценка результата:**
- Легко 🟢 → stage+1, интервал × 1.3
- Помню 🟡 → stage+1, интервал без изменений
- Сложно 🔴 → stage не меняется, интервал × 0.6 (минимум 1 день)
- В fill-in-the-blank: правильно = «Помню», неправильно = «Сложно»

**Пропущенные слова (overdue):** если `next_review_date < сегодня`, слово переходит в пул `overdue`, stage откатывается на 1 назад. В `/review` показываются первыми с пометкой ⚠️.

**Защита от повтора в сессии:** `context.user_data["review_shown"]` — set ID слов, показанных в текущей сессии /review.

## Команды бота
| Команда | Что делает |
|---------|-----------|
| Любой текст | Добавить слово: Claude объясняет, сохраняется в БД |
| `/words` | Подобрать 5 новых слов через Claude (адаптируется под уже добавленные) |
| `/review` | Повторение по расписанию (сначала overdue, потом scheduled) |
| `/all` | Повторить все слова из базы в случайном порядке |
| `/delete` | Удалить слово (с аргументом или через список кнопок) |
| `/stats` | Статистика по статусам и уровням A1/A2 |

## Типы карточек при повторении
1. **Recognition** — показывается русский перевод, нужно вспомнить испанское слово. Кнопка «Показать ответ» → три кнопки оценки.
2. **Fill-in-the-blank** — испанское предложение с пропуском, выбор из 3 вариантов (правильное слово + 2 случайных из базы). Используется если есть примеры и достаточно слов для дистракторов.

Чередование: fill-in-the-blank приоритетнее, fallback на recognition.

## Уведомления
- **10:00 МСК (07:00 UTC)** — утреннее, только если есть слова не повторённые сегодня
- **18:00 МСК (15:00 UTC)** — вечернее, только если есть слова не повторённые сегодня
- Текст: «☀️ Доброе утро! Слов на повторение: N» / «🌙 Добрый вечер! Слов на повторение: N»

## Статусы слов
| Статус | Ступень | Смысл |
|--------|---------|-------|
| collected | 0 | Добавлено, ещё не повторялось |
| learning | 1–2 | В процессе изучения |
| familiar | 3–4 | Знакомо |
| active | 5 | Активно используется |
| mastered | 6 | Выучено |
| skipped | — | Пропущено в /words (не попадает в повторение) |

## Ключевые решения и ограничения
- **Существительные всегда с артиклем** — `la casa`, `el coche`. Это требование к промпту, Claude всегда должен возвращать с артиклем.
- **Глаголы всегда со спряжением** — поле `conjugation` заполняется только для глаголов.
- **/words пропускает слова**, уже есть в базе (включая skipped). Список передаётся в промпт.
- **Пропущенные слова** сохраняются в БД со статусом `skipped` — Claude их не предложит повторно.
- **context.user_data** хранит состояние очередей (/words, /all) и set показанных слов — сбрасывается при перезапуске бота.
- **Деплой на Railway** в том же проекте что и английский vocab_bot (vocab-bot). Volume примонтирован на `/data`.
