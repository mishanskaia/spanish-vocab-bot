# CLAUDE.md — Руководство для нового ИИ-разработчика

Этот файл написан так, чтобы ИИ без какого-либо предыдущего контекста мог продолжить разработку.
Читай внимательно: здесь объясняется не только ЧТО сделано, но и ПОЧЕМУ.

---

## Что это за проект

Telegram-бот для изучения испанского языка с нуля. Пользователь — русскоязычный, уровень A1-A2.
Бот хранит словарный запас и повторяет слова по кривой интервальных повторений.

Это самостоятельный проект. Существует также похожий английский бот (`../vocab_bot/`) —
он послужил архитектурной основой, но это разные боты с разными БД и разными токенами.

```bash
pip install -r requirements.txt
# создай .env на основе .env.example, заполни токены
```

Railway.app, сервис `spanish-vocab-bot` внутри проекта `vocab-bot` (там же живёт английский бот).
Persistent Volume примонтирован на `/data` — без него БД сбрасывается при каждом рестарте.
Деплой автоматический при пуше в ветку `main` на GitHub: `mishanskaia/spanish-vocab-bot`.

## Архитектура: три файла

### `bot.py` — всё что связано с Telegram
Хендлеры команд, кнопок, сообщений. Логика отображения карточек повторения.
Состояние очередей хранится в `context.user_data` — **эфемерно**, теряется при рестарте бота.

### `db.py` — вся работа с SQLite
Схема таблицы, CRUD, логика интервалов, определение окна суток, пулы overdue/scheduled.
**Не держи бизнес-логику интервалов в bot.py** — всё в db.py.

### `ai_helper.py` — два промпта к Claude Sonnet 4.6
- `explain_word(word)` — объяснить одно слово
- `find_frequent_words(existing_words, count)` — подобрать N новых слов

Claude отвечает только JSON. Парсинг в `_ask_claude()` — стрипает markdown-обёртку если есть.

## Схема БД (таблица `words`)

```sql
id, user_id, phrase, meaning, part_of_speech, cefr_level,
examples (JSON-массив строк),
conjugation (строка или NULL, только для глаголов),
added_date, interval_stage (0–6), next_review_date,
correct_streak, status, times_reviewed, success_rate, last_reviewed,
pool ('scheduled'|'overdue'),
added_window ('morning'|'evening')
```

**Важные инварианты:**
- Существительные в `phrase` — всегда с артиклем: `la casa`, `el coche`. Это требование к промпту.
- Глаголы в `phrase` — инфинитив: `hablar`, `ser`. Спряжение отдельно в `conjugation`.
- `examples` — JSON-массив, хранится как TEXT. При чтении всегда `json.loads(row["examples"] or "[]")`.
- `status = 'skipped'` — слово пропущено в /words. Не участвует в повторениях, но передаётся Claude чтобы не предлагал повторно.
- `pool = 'overdue'` — слово не было повторено в срок. Устанавливается в `detect_and_mark_overdue()`.

## Логика интервальных повторений

```python
INTERVALS = [1, 3, 7, 14, 30, 90]  # дней, 6 ступеней
```

**Первое повторение** — в тот же день или на следующий, зависит от окна суток (МСК):
- Утро (6–14): `next_review_date = сегодня`, `added_window = 'morning'`
- Вечер (14–23) или ночь: `next_review_date = завтра`, `added_window = 'evening'`

**Оценка при повторении** (`mark_review_result(word_id, grade)`):
- `'easy'`: `stage += 1`, дней = `ceil(INTERVALS[stage] * 1.3)`
- `'remember'`: `stage += 1`, дней = `INTERVALS[stage]`
- `'hard'`: `stage` не меняется, дней = `max(1, floor(INTERVALS[stage] * 0.6))`

После stage 5 (последнего) → `status = 'mastered'`.

**Overdue:** если `next_review_date < сегодня` и `pool = 'scheduled'` → `pool = 'overdue'`, `stage -= 1`.
Вызывается в `detect_and_mark_overdue(user_id)` при старте каждой сессии /review.

## Пользовательский флоу

### Добавление слова
Пользователь пишет любой текст (не команду) → `handle_message()` → Claude → сохранить в БД.
**Нет команды /add** — это сознательное решение для простоты UX. Не добавляй /add без нужды.

### /words
Подбирает 5 новых слов через Claude. Передаёт список ВСЕХ слов пользователя (включая skipped)
чтобы Claude не повторялся. Слова показываются по одному с кнопками Сохранить/Пропустить.
Пропущенные сохраняются в БД со статусом `skipped`.

Было 10 слов — уменьшено до 5 из-за таймаута Claude (генерация 10 слов со спряжениями занимает >60 сек).

### /review
1. Очищает `context.user_data["review_shown"]` (set ID уже показанных в этой сессии)
2. Вызывает `detect_and_mark_overdue()`
3. Показывает overdue-слова первыми (с ⚠️), потом scheduled
4. Тип карточки: fill-in-the-blank если есть примеры и ≥2 дистракторов, иначе recognition
5. Fill-in-the-blank: правильно = grade 'remember', неправильно = grade 'hard'
6. Recognition: три кнопки — Легко 🟢 / Помню 🟡 / Сложно 🔴

### /all
Повторить все слова из базы в случайном порядке, без учёта расписания.
Использует отдельную очередь `context.user_data["all_queue"]`.
После grade — переходит к следующему в очереди, не перезапрашивает БД.

---

## Типы карточек при повторении

**Recognition (самооценка):**
Показывает: `Как будет по-испански: *{meaning}*`
→ кнопка «Показать ответ»
→ показывает `{phrase}` с примерами и спряжением
→ три кнопки оценки

**Fill-in-the-blank (multiple choice):**
Берёт первый пример из `examples`, ищет слово (без артикля для существительных) в тексте,
заменяет на `_____`. Кнопки: правильное слово + 2 случайных дистрактора из базы пользователя.
Если пример не содержит слово → fallback на recognition.

---

## Уведомления

- **10:00 МСК (07:00 UTC)** — `morning_reminder()`: "☀️ Доброе утро! Слов на повторение: N"
- **18:00 МСК (15:00 UTC)** — `evening_reminder()`: "🌙 Добрый вечер! Слов на повторение: N"
- Отправляются только если есть слова, не повторённые сегодня (`count_due_not_reviewed_today()`)
- После уведомления сразу начинается сессия повторения

## Статусы слов

| status | stage | Смысл |
|--------|-------|-------|
| collected | 0 | Добавлено, ещё ни разу не повторялось |
| learning | 1–2 | Активно учим |
| familiar | 3–4 | Знакомо |
| active | 5 | Уверенно |
| mastered | 6 | Выучено, не показывается в /review |
| skipped | — | Пропущено в /words, не участвует в повторениях |

## Известные проблемы и ограничения

1. **context.user_data эфемерно** — очереди /words и /all, а также `review_shown` теряются при рестарте бота на Railway. Пользователь должен начать заново. Для фикса нужно хранить состояние в БД.

2. **/words может быть медленным** — Claude генерирует 5 слов с примерами и спряжениями, занимает 15–30 сек. Бот показывает "Подбираю слова, подожди немного..." но второй индикатор прогресса нет.

3. **Fill-in-the-blank часто не срабатывает** — если пример не содержит точного вхождения слова (другая форма глагола, множественное число существительного и т.д.), падает на recognition. Это нормально, recognition — хорошая карточка.

4. **Слово может не совпадать с примером** — Claude иногда возвращает пример где слово стоит в другой форме. fill-in-the-blank тогда не находит пропуск → fallback. Не критично.

5. **Overdue-пул не сбрасывается автоматически** — если пользователь не заходит несколько дней, слова накапливаются в overdue. При следующем /review все придут сразу. Это намеренно.

## Что планировалось но не сделано

- **Озвучка слов** — голосовое сообщение с произношением. Отложено до фазы immersion (требует TTS-сервис).
- **Словосочетания/коллокации** — отдельное поле. Не нужно на A1-A2, добавить позже по аналогии с английским ботом.
- **Фаза /topic** — подбор слов по теме (как в английском боте). Пока не реализовано.
- **Хранение состояния сессий в БД** — чтобы не терять очередь при рестарте.
